# 安全策略

AgentShield MCP 本身是一个安全工具，我们对它自己的安全性同样认真。

## 报告漏洞

**请不要**通过公开 Issue 报告安全漏洞。

请发送邮件到 **maorongkang@gmail.com**，主题以 `[SECURITY]` 开头，内容尽量包含：

- 漏洞类型与影响范围
- 复现步骤或 PoC
- 受影响的版本 / commit
- 你认为可行的修复方向（可选）

我们会在 **3 个工作日内**确认收到，并在评估后与你协商披露时间。修复发布前请勿公开细节。

## 支持的版本

项目处于 MVP 阶段，仅对最新的 `main` 分支和最近一个发布版本提供安全修复。

| 版本 | 安全更新 |
|---|---|
| 最新 release | 支持 |
| 旧版本 | 不支持，请升级 |

## 安全设计约定

这些是项目自身必须坚持的安全基线，贡献代码时请遵守：

- **默认本地运行**，不向任何外部服务上传用户数据（SEC-001）。
- 审计日志中的敏感字段（Token、密码、密钥）必须**脱敏**后存储（SEC-003 / SEC-007）。
- 终端、报告、日志中**不得明文展示**完整 Token（SEC-002）。
- 默认规则**阻止**读取 `.env`、SSH 私钥、`*.pem` 等敏感文件（SEC-004）。
- 默认规则**阻止** `curl | bash` 等远程脚本直接执行（SEC-005）。
- 配置文件（`.agentshield/`）权限应限制为当前用户可读写（SEC-006）。

## 威胁模型边界

AgentShield 防护的是“AI Agent 经由 MCP 发起的工具调用”这一层。以下不在防护范围内，请勿据此产生错误的安全预期：

- 绕过 MCP、由其他本地进程直接发起的操作
- 用户手动在终端执行的命令
- 恶意 MCP Server 在被放行后自身的行为
- 操作系统层面的提权与持久化

详见 README 的「它不做什么」一节。
